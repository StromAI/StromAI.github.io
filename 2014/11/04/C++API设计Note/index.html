C++API设计Note

整理自《C++ API设计》一书中的提示，不定期更新

1.特征

  • 避免将函数声明为可以重写的（虚）函数，除非你有合理而迫切的需求
  • 基于最小化的核心API，以独立的模块或苦地行使构建便捷API
  • 使用枚举类型代替布尔类型，可以提高代码的可读性
  • 避免编写拥有多个相同类型参数的函数
  • 使用一致的函数命名和参数顺序
